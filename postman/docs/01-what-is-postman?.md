# Purpose

This guide introduces Postman, explains what it is used for, and outlines the types of tasks it helps users perform. After reading this guide, you should understand why Postman is widely used by developers and know what to expect as you continue through this documentation set.

---

# What is Postman?

Postman is an API platform that helps developers design, test, document, and manage APIs throughout the software development lifecycle.

Instead of writing code to manually communicate with an API, you can use Postman to create requests, inspect responses, organize your work, and collaborate with others using a graphical interface.

Postman supports individual developers, development teams, and enterprise organizations by providing tools for API development, testing, collaboration, and governance.

---

# Why use Postman?

Applications constantly exchange information with other applications through APIs.

Postman provides a single platform where you can:

- Build and send API requests.
- Inspect API responses.
- Organize requests into reusable collections.
- Test API behaviour.
- Share API resources with teammates.
- Document APIs.
- Monitor API performance.

By bringing these capabilities together, Postman simplifies many common API development tasks.

## Why use Postman?

Modern applications constantly exchange information with other applications through APIs. Postman brings the tools needed to work with those APIs into a single platform.

With Postman, you can:

- Send API requests.
- Inspect API responses.
- Organize requests into Collections.
- Test API behaviour.
- Generate API documentation.
- Collaborate with teammates using shared Workspaces.
- Monitor APIs.
- Work with multiple API protocols.

---

## Key capabilities

Postman supports different stages of the API lifecycle.

Some of its core capabilities include:

- Designing and testing APIs.
- Creating reusable Collections.
- Managing Workspaces for collaboration.
- Running automated tests.
- Publishing API documentation.
- Monitoring API performance.
- Supporting multiple protocols, including HTTP, GraphQL, gRPC, WebSockets, MQTT, and others.
- AI-assisted workflows for building and testing APIs.

Depending on your role, you may only use a subset of these capabilities.
