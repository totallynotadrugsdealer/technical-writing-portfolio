# Documentation Opportunities

It does not explain how a beginner can start using it.

Possible beginner documentation:

- What is Postman?
- Installing Postman
- Understanding APIs
- Understanding Workspaces
- Sending Your First Request
- Understanding Collections
- Understanding Responses
- Troubleshooting Common Errors

# Observations on Writing Style

Homepage copy characteristics:

- Very concise.
- Marketing-oriented.
- Benefits before features.
- Heavy use of short paragraphs.
- Uses cards instead of long text.
- Uses audience segmentation (Developers, Teams, Enterprise).
- Minimal technical explanation.
- Assumes familiarity with API terminology.

Implication for this documentation:

Documentation should explain concepts the homepage assumes users already understand.
