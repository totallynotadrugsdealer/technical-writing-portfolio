# Style Guide

## Purpose

This style guide defines the writing standards used throughout the Postman documentation project.

The goal is to produce documentation that is clear, consistent, accurate, and approachable for beginners learning APIs and Postman for the first time.

---

## Audience

This documentation is written for beginners with little or no experience working with APIs.

Assume readers are comfortable using computers but unfamiliar with API terminology.

---

# Writing Principles

- Prioritize clarity over technical complexity.
- Explain unfamiliar terminology before using it.
- Keep instructions concise.
- Write from the user's perspective.
- One topic per document.

---

# Voice and Tone

Voice:
Professional

Tone:
Friendly, encouraging, and instructional.

Avoid humour, slang, and unnecessary jargon.

---

# Point of View

Write in the second person.

✔ You can send a request.

✘ The user can send a request.

---

# Tense

Use present tense whenever possible.

✔ Postman stores requests in Collections.

✘ Postman will store requests in Collections.

---

## Headings

Use sentence case for all headings.

Headings should describe the action or concept rather than being overly general.

✔ Create your first request

✔ Understanding Collections

✘ Tutorial

✘ Guide

✘ Examples

---

## Procedures

Write instructions as numbered steps.

Begin each step with an imperative verb.

Each step should describe one action only.

✔ 1. Select **Send**.

✔ 2. Review the response body.

✘ Click Send and then look at the response while making sure everything works.

---

# User Interface Elements

Use bold formatting for buttons, menu items, and interface labels.

Example:

Select **Send**.

Open **Collections**.

---

# Notes

Use blockquotes.

> Note:
> You must be connected to the internet to sign in.

---

# Screenshots

Include screenshots only when they clarify an action or interface.

Avoid screenshots that contain unnecessary UI elements.

---

# Code

Use fenced code blocks.

Specify the language whenever possible.

---

# Terminology

Use terms consistently.

Always capitalize:

- Postman
- Collection
- Workspace
- Environment

Avoid abbreviations unless previously explained.
